# Phase 1 — UI Design Contract: Project Foundation

> Visual and interaction contract for the RBLaunch design system foundation. This phase produces no visible pages — it defines the token system, typography scale, color palette, component primitives, motion principles, and logo rules that every downstream phase consumes. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom components) |
| Preset | not applicable |
| Component library | none — hand-rolled for full brand control |
| Icon library | none (no icons needed in Phase 1; evaluate Lucide in Phase 2 if needed) |
| Font heading | Barlow Condensed via `next/font/google` (weight 700) |
| Font body | Inter via `next/font/google` (weight 400) |
| CSS framework | Tailwind CSS v4 — CSS-first `@theme` config in `globals.css` |
| Animation | motion 12.x — import from `motion/react` |

**Design philosophy:** Dark-first. The site is predominantly black (#000000) backgrounds with white text. This is a bold, high-contrast agency aesthetic — not a "dark mode toggle" system.

## Tailwind v4 @theme Token System

All tokens defined once in `src/app/globals.css` using `@theme`. No `tailwind.config.js` file.

### Color Tokens

```css
@theme {
  /* Brand palette */
  --color-black: #000000;
  --color-white: #FFFFFF;
  --color-red: #E8232A;
  --color-dark-grey: #111111;
  --color-muted: #888888;

  /* Semantic aliases */
  --color-background: #000000;
  --color-surface: #111111;
  --color-foreground: #FFFFFF;
  --color-foreground-muted: #888888;
  --color-accent: #E8232A;
  --color-accent-hover: #CC1E24;

  /* Font families */
  --font-heading: var(--font-barlow);
  --font-body: var(--font-inter);

  /* Border radius */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --radius-full: 9999px;

  /* Shadows */
  --shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.4);
  --shadow-md: 0 4px 12px rgba(0, 0, 0, 0.5);
  --shadow-lg: 0 8px 24px rgba(0, 0, 0, 0.6);
}
```

### Font CSS Variable Setup

In `src/lib/fonts.ts`:
```typescript
import { Barlow_Condensed, Inter } from 'next/font/google'

export const barlow = Barlow_Condensed({
  subsets: ['latin'],
  weight: ['700'],
  variable: '--font-barlow',
  display: 'swap',
})

export const inter = Inter({
  subsets: ['latin'],
  weight: ['400', '700'],
  variable: '--font-inter',
  display: 'swap',
})
```

In `src/app/layout.tsx`, apply both variables to `<html>`:
```tsx
<html className={`${barlow.variable} ${inter.variable}`}>
```

## Typography

### Type Scale

4 canonical sizes. Responsive scaling between these values is an implementation detail handled via Tailwind responsive prefixes.

| Size | Font | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| 14px | Inter | 400 | 1.5 | Small text, captions, labels, nav links |
| 16px | Inter | 400 | 1.6 | Body text, buttons |
| 32px | Barlow Condensed | 700 | 1.15 | Subheadings (H3, H4) |
| 56px | Barlow Condensed | 700 | 1.05 | Display headings (H1, H2) |

### Declared Weights

| Weight | Font | Usage |
|--------|------|-------|
| 400 | Inter | Body text, small text, captions, labels, nav links |
| 700 | Barlow Condensed headings, Inter bold emphasis | All headings (H1-H4), buttons, taglines, bold inline text |

### Typography Rules

1. Headings (H1-H3) are ALWAYS uppercase. Barlow Condensed's condensed width makes uppercase readable without feeling shouty.
2. Body text is NEVER uppercase. Inter body stays sentence case.
3. Maximum body text width: 640px (40ch). Prevents unreadable long lines on desktop.
4. Heading sizes scale between mobile and desktop using Tailwind responsive prefixes (`text-3xl md:text-5xl`), not `clamp()`. The 4 canonical sizes above are the declared scale; intermediate responsive values are implementation details.
5. Buttons use Inter at 16px, weight 700, uppercase, letter-spacing 0.02em.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #000000 (black) | Page background, hero background, major section backgrounds |
| Secondary (30%) | #111111 (dark grey) | Card surfaces, alternate section backgrounds, elevated surfaces |
| Accent (10%) | #E8232A (deep red) | See reserved list below |
| Foreground primary | #FFFFFF (white) | All heading text, body text on dark backgrounds |
| Foreground muted | #888888 (muted grey) | Secondary text, captions, dividers, placeholder text |
| Accent hover | #CC1E24 (darker red) | Hover/active state for accent elements |

### Accent Reserved For (exhaustive list)

The deep red (#E8232A) is used ONLY on these elements — never on backgrounds, never on body text:

1. Primary CTA button background ("Book a Free Call")
2. Active/hover underline on nav links
3. Step numbers in "How It Works" section
4. The vertical accent bar on service cards (left border)
5. Social media icon hover state
6. FAQ accordion expand indicator (the "+" icon)
7. Inline text links on hover

### Light Section Exception

If any section uses a white/light background (e.g., a proof strip):
- Heading text: #000000 (black)
- Body text: #111111 (dark grey)
- Accent remains: #E8232A (unchanged)
- Logo variant: horizontal-light (black logo)

## Logo Usage Rules

| Variant | File | Usage Context |
|---------|------|---------------|
| Horizontal light | `horizontal-light.svg` or `.png` | Light backgrounds only — black logo on white/light surface. Used in: light nav bar (if scrolled to light section) |
| Horizontal dark | `horizontal-dark.svg` or `.png` | Dark backgrounds — white logo on black/dark surface. Used in: sticky header (default), hero, dark sections, footer |
| Square icon | `icon-square.svg` or `.png` | Favicon (`app/icon.png`), social media profile photos, OG image brand mark, Apple touch icon |

### Logo Sizing

| Context | Height | Notes |
|---------|--------|-------|
| Sticky header | 32px | Fixed height, width scales proportionally |
| Footer | 40px | Slightly larger for visual hierarchy |
| Favicon | 32x32 | Square icon variant |
| Apple touch icon | 180x180 | Square icon variant |
| OG image | 80px height | Square icon variant, centered in 1200x630 |

### Logo Rules

1. Never stretch or distort — always maintain aspect ratio
2. Minimum clear space around logo: 16px (1x the spacing `md` token)
3. Never place the light variant on a dark background or vice versa
4. Import as Next.js Image component for automatic optimization

## Motion / Animation Principles

### Global Rules

1. **NEVER animate the hero headline** — `initial={{ opacity: 0 }}` on the H1 kills LCP. The hero heading must render immediately with full opacity.
2. **Only animate below-fold elements** — elements that start outside the viewport.
3. **Prefer `whileInView` over scroll-linked** — simpler, more performant for entrance animations.
4. **No animation on reduced-motion** — respect `prefers-reduced-motion: reduce`. All motion wrappers must check this.

### Standard Entrance Animation (FadeUp)

Used for all below-fold content: section headings, cards, text blocks.

| Property | Value |
|----------|-------|
| Initial | opacity: 0, y: 24px |
| Animate | opacity: 1, y: 0 |
| Duration | 0.5s (500ms) |
| Easing | [0.25, 0.1, 0.25, 1.0] (ease-out cubic) |
| Viewport trigger | `whileInView`, `once: true`, `amount: 0.2` |

### Stagger Children

Used for card grids, step sequences, FAQ items.

| Property | Value |
|----------|-------|
| Stagger delay | 0.1s (100ms) between children |
| Container | `variants` with `staggerChildren: 0.1` |

### Button Hover

| Property | Value |
|----------|-------|
| Transform | translateY(-1px) |
| Duration | 150ms |
| Easing | ease |

### Page Transitions

None. Single-page app with anchor scrolling — no page transition animations.

### What NOT to Animate

- Hero headline (LCP killer)
- Logo in header (distracting)
- Navigation links (feels sluggish)
- Text that is already in viewport on load
- Background colors (performance cost, no UX benefit)
- Font size or weight (causes layout thrashing)

## Responsive Breakpoints

| Name | Min width | Usage |
|------|-----------|-------|
| mobile (default) | 0px | Base styles, single column |
| sm | 640px | Minor adjustments |
| md | 768px | Tablet — 2-column grids, larger typography |
| lg | 1024px | Small desktop |
| xl | 1280px | Desktop — max content width, full typography scale |

### Content Constraints

| Element | Max width | Centered |
|---------|-----------|----------|
| Page content wrapper | 1280px | Yes (mx-auto) |
| Text content (body paragraphs) | 640px | Yes within section |
| Card grid | 1120px | Yes within page wrapper |
| Header inner | 1280px | Yes |

### Mobile-First Rules

1. All styles are written mobile-first. Desktop overrides use `md:` and `xl:` prefixes.
2. Section vertical padding: 48px mobile, 64px tablet, 96px desktop.
3. Card grid: 1 column mobile, 2 columns at `md:`.
4. Navigation: hamburger menu at mobile, inline links at `md:`.

## Global CSS Reset / Base Styles

```css
/* Base layer */
@layer base {
  html {
    scroll-behavior: smooth;
    -webkit-font-smoothing: antialiased;

  }

    font-family: var(--font-heading);

    line-height: 1.1;

    scroll-margin-top: 80px;
  }

  /* Focus visible for keyboard navigation */

  }

  /* Selection color */
  ::selection {
    background-color: var(--color-accent);
    color: var(--color-foreground);
  }
}
```

## Accessibility Contract

1. Color contrast: #FFFFFF on #000000 = 21:1 (exceeds WCAG AAA). #FFFFFF on #111111 = 18.3:1 (exceeds AAA). #888888 on #000000 = 5.3:1 (passes AA for large text, AA for normal text).
2. All interactive elements have visible `:focus-visible` outlines (2px solid white, 2px offset).
3. All buttons have minimum 44px touch target height.
4. Motion respects `prefers-reduced-motion: reduce`.
5. Images include descriptive `alt` attributes (logo alt: "RBLaunch").
6. Semantic HTML: `<header>`, `<main>`, `<section>`, `<footer>`, `<nav>`.
